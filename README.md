# Linkly // Premium URL Shortener & Link Analytics Service (Spring Boot)

Linkly is a highly optimized URL Shortener and Link Analytics service built using **Spring Boot** and Java 21+. It utilizes Spring Web REST Controllers, handles JSON parsing natively using Jackson, and manages dependencies via Maven. It retains its custom thread-safe CSV-based transactional file database with atomic write guarantees.

The service comes with a **premium, glassmorphic dark-mode web dashboard** served directly at the root URL (`http://localhost:8080/`), allowing users to easily shorten links, set custom aliases, and inspect analytics.

---

## Technical Features

1. **Clean Spring MVC Architecture**: Implements `@RestController` for API handling (`/shorten` and `/analytics/{code}`) and a `@Controller` for capturing dynamic `/{code}` path variables to issue redirects.
2. **Bijective Base62 Short Codes**: Generated by Base62-encoding an atomic auto-increment counter. The mapping is bijective, ensuring zero collisions across restarts and concurrent operations.
3. **Duplicate URL Handling**: Idempotent deduplication. Shortening the same URL twice returns the same short code. Custom aliases are treated as independent paths, so they always generate a fresh mapping to honor the custom request.
4. **URL Validation**: Ensures only valid `http` or `https` URLs are shortened. Rejects malformed URIs, and actively prevents shortening loops (where the destination URL matches the shortener's own domain and port).
5. **Robust Concurrency**: All database operations are guarded by a `ReentrantReadWriteLock`. File writes are serialized, and transient Windows-specific file locks are handled gracefully using thread-unique temporary files and file move retry loops with backoff.

## Installation & Requirements

- **Java Development Kit (JDK)**: Version 21 or higher is required (Java 25 is pre-configured in this repository).
- **Environment**: No pre-installed build tools (Maven, Gradle) are required. The project includes a self-contained Maven Wrapper (`mvnw.cmd` and `.mvn/`).

## API Documentation

### 1. Shorten URL (`POST /shorten`)
Accepts JSON payloads.

#### Request (JSON)
- **Headers**: `Content-Type: application/json`
- **Body**:
  ```json
  {
    "url": "https://github.com/google/gemini-client",
    "alias": "gemini-client"
  }
  ```

#### Request (cURL Example)
```bash
curl -X POST http://localhost:8080/shorten \
  -H "Content-Type: application/json" \
  -d '{"url": "https://github.com/google/gemini-client", "alias": "gemini-client"}'
```

#### Response (`201 Created`)
```json
{
  "short_url": "http://localhost:8080/gemini-client",
  "code": "gemini-client",
  "original_url": "https://github.com/google/gemini-client",
  "is_custom": true,
  "created_at": 1781204257000,
  "clicks": 0
}
```

---

### 2. Redirect URL (`GET /{code}`)
Redirects the client to the destination.

#### Request (cURL Example)
```bash
curl -i http://localhost:8080/gemini-client
```

#### Response (`301 Moved Permanently`)
```http
HTTP/1.1 301 Moved Permanently
Location: https://github.com/google/gemini-client
Content-Length: 0
```
*(If the code does not exist, serves a custom, beautifully styled 404 HTML page).*

---

### 3. Link Analytics (`GET /analytics/{code}`)
Fetches real-time click statistics and metadata for a short code.

#### Request (cURL Example)
```bash
curl http://localhost:8080/analytics/gemini-client
```

#### Response (`200 OK`)
```json
{
  "code": "gemini-client",
  "original_url": "https://github.com/google/gemini-client",
  "is_custom": true,
  "created_at": 1781204257000,
  "clicks": 1
}
```
